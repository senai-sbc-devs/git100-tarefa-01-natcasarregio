Ciclo de fluxo de trabalho do Git e GitHub

Descrição dos comandos "Clone", "Branch", "Commit", "Merge" e "Pull Request".

Git Clone

O comando baixa o código-fonte de um repositório remoto, fazendo uma cópia idêntica da versão mais recente do um projeto e a salvando em seu espaço de trabalho local.

Git Branch

Branches (ou ramificações) possibilitam que vários desenvolvedores trabalhem em paralelo no mesmo projeto ao mesmo tempo. Com o comando, é possível criar, listar e excluir branches.

Git Commit

O comando salva as alterações do código, definindo um ponto de verificação no processo de desenvolvimento; ainda é possível escrever uma mensagem breve para explicar o que foi desenvolvido no código-fonte.

Git Merge

É usado na etapa final de desenvolvimento, que une a branch atual com a main. Dessa forma, ele integra todos os seus commits na branch de desenvolvimento ou na branch principal.

Pull Request

É uma proposta para fazer merge de alterações feitas em uma ramificação de um repositório local em outra, normalmente de uma ramificação de recursos para a ramificação principal.
